# BRAND_DIRECTION.md — EstacionAR x Municipalidad de Salta

## Nombre
- Mantener el producto como **EstacionAR**.
- Usar co-branding institucional: **EstacionAR · Municipalidad de Salta**.
- No renombrar el sistema a “Salta”; Salta es el organismo/cliente.

## Referencia visual institucional
Imagen de referencia: `brand_refs/municipalidad_salta.jpeg`.

Lectura de marca:
- Logo/texto visible: “Salta” + “MUNICIPALIDAD”.
- Estilo: limpio, institucional, moderno, alto contraste.
- Azul institucional aproximado: `#0067B1` / `#006BB6`.
- Fondo institucional: blanco/gris muy claro `#F5F5F5`.

## Dirección visual integrada
Combinar identidad municipal con MotionSite premium EstacionAR:
- Base institucional clara: blanco, gris claro, azul municipal.
- Capa tecnológica premium: azul noche `#0A1A2F`, azul profundo `#102A47`, cyan eléctrico `#0FB6CE`.
- Acento vial: ámbar `#F5A623` para estacionamiento, alertas suaves, calles/sectores.

## Uso de marca en UI
- Header/hero: “EstacionAR” principal + badge “Municipalidad de Salta”.
- Splash/login: aplicar co-branding institucional.
- Backoffice: barra superior con “Municipalidad de Salta · Control de estacionamiento medido”.
- Comprobantes: incluir “Municipalidad de Salta” como organismo recaudador.
- App permisionario: credencial/QR con referencia municipal.
- Footer: “Sistema de estacionamiento medido para la Municipalidad de Salta”.

## No hacer
- No cambiar el nombre EstacionAR.
- No deformar ni reinventar el logo municipal.
- No usar estética genérica SaaS/crypto.
- No esconder el rol fiscal municipal: debe verse que la recaudación entra al Municipio.
